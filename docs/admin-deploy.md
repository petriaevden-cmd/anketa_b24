# Анкета · Админ-панель — Деплой

Standalone-страница для редактирования **любых UF-полей лида** в Битрикс24.
Живёт по тому же пути, что и `index.php` (рядом с анкетой), отдельным файлом
`admin.php`. Путь зависит от вашего хоста:

- `https://yurclick.com/anketa/admin.php` (prod)
- `https://apcheit.ru/yurclick/anketa-kc/admin.php` (или ваш вариант)

---

## Что добавлено

```
admin.php              ← страница входа + UI (HTML + Tailwind CDN)
admin-api.php          ← REST-endpoint для AJAX (search/get/save/log)
admin-config.php       ← константы (пароль, лог-файл, карта UF-полей)
assets/admin.js        ← клиентская логика (поиск, рендер, сохранение)
admin-changes.log      ← создаётся автоматически при первой правке
docs/DEPLOY.md         ← этот файл
```

`admin.php` и `admin-config.php` можно безопасно коммитить в публичный репо —
в них нет секретов. `config.php` (там лежит `WEBHOOK_URL`) как был в `.gitignore`,
так и остаётся.

## Деплой

### 1. Положить файлы

Скопировать **четыре** файла в ту же папку, где лежит `index.php`:

```
<корень-анкеты>/admin.php
<корень-анкеты>/admin-api.php
<корень-анкеты>/admin-config.php
<корень-анкеты>/assets/admin.js
```

Примеры корня:
- `https://yurclick.com/anketa/` → файлы в `/anketa/`
- `https://apcheit.ru/yurclick/anketa-kc/` → файлы в `/yurclick/anketa-kc/`

Это новые файлы, ничего не заменяют.

### 2. Проверить config.php

В `config.php` рядом с `WEBHOOK_URL` ничего менять не нужно — админка использует
тот же вебхук. Если у вас несколько порталов (prod/dev), `WEBHOOK_URL` уже должен
выбираться по `DOMAIN` (как в `index.php`).

### 3. Права на лог-файл

`admin-changes.log` создаётся автоматически в той же папке. Если PHP работает
от отдельного пользователя, дать права на запись:

```bash
chmod 775 <корень-анкеты>
touch <корень-анкеты>/admin-changes.log
chmod 666 <корень-анкеты>/admin-changes.log
```

### 4. Открыть и проверить

Перейти на `https://<ваш-домен>/<путь>/admin.php` и войти с паролем `admin`.
Должны увидеть список последних лидов.

## Смена пароля

Сейчас пароль = `admin` (см. `admin-config.php`, константа `ADMIN_PASSWORD`).
**Поменять перед боевым использованием.**

Самый простой способ — открыть `admin-config.php` и поменять значение:

```php
define('ADMIN_PASSWORD', 'тут_сложный_пароль_минимум_16_символов');
```

Загрузить файл обратно. Старая сессия (если была открыта) автоматически
протухнет через `ADMIN_SESSION_LIFETIME` (по умолчанию 8 часов).

## Поток работы

1. Открыть `https://<ваш-домен>/<путь>/admin.php`
2. Ввести пароль → вход
3. В поиске ввести ID, фамилию или телефон → Enter
4. Кликнуть на лид в списке → откроется редактор
5. Поменять поля, нажать **«Сохранить изменения»** (или Ctrl+S)
6. Все изменения запишутся в `admin-changes.log` (видно внизу справа)
7. Кнопка **«Открыть в CRM ↗»** ведёт в карточку лида в Битриксе

## Безопасность

- **Пароль в открытом виде** в `admin-config.php`. Это компромисс ради простоты.
  Для production-замены: подключить `password_hash()` / `password_verify()` и
  хранить хеш в `config.php` рядом с `WEBHOOK_URL`.
- **Все действия логируются** в `admin-changes.log`. Видно кто/когда/что
  изменил.
- **Сессия админа** живёт 8 часов (настраивается через `ADMIN_SESSION_LIFETIME`).
- **CSRF-защита** для `save` сейчас не реализована — для параноиков можно
  добавить одноразовый токен в форму. Но раз админка уже под паролем и доступ
  только по логину/паролю с прямого URL — для MVP достаточно.
- **Спрятать URL** — можно переименовать `admin.php` в любую длинную строку
  (`x8s2k-admin.php` и т.п.). Главное чтобы параллельно переименовался и путь
  в `admin.js` (`API_URL` строится динамически — должен подхватить).

## Как добавить новое поле в админку

Открыть `admin-config.php`, найти массив `$ADMIN_FIELD_MAP` и добавить запись:

```php
'UF_CRM_НОВЫЙ_КОД' => [
    'label' => 'Человекочитаемая метка',
    'group' => 'Название группы',
    'type'  => 'string',  // или money / boolean / enum / datetime / readonly
    'hint'  => 'Подсказка под полем (опционально)',
],
```

Для enum-поля добавить `options`:

```php
'UF_CRM_...' => [
    'label' => '...',
    'group' => '...',
    'type'  => 'enum',
    'options' => [
        100 => 'Вариант 1',
        200 => 'Вариант 2',
    ],
],
```

Поле сразу появится в UI. Никакой миграции не нужно.

## Лог изменений

Файл `admin-changes.log` (рядом с `index.php`) — JSON-Lines, по одной записи
на строку:

```json
{"ts":"2026-06-18T12:30:45+04:00","user":"admin","lead":12345,"field":"UF_CRM_1764765025374","old":"Иван","new":"Иван Иванович"}
```

Посмотреть последние 20 правок в одном лиде:

```bash
grep '"lead":12345' admin-changes.log | tail -20
```

Скачать целиком: `https://<ваш-домен>/<путь>/admin-changes.log`
(если хост отдаёт статику из папки — иначе настроить).

## Что делать, если сломалось

| Симптом | Что проверить |
|---|---|
| `Не удалось получить карту полей: HTTP 200: сервер вернул не-JSON` | admin.js ходит на admin-api.php, а не на admin.php. Обновить `assets/admin.js` (исправлено в commit f2f1230) |
| `Неверный пароль` сразу | Сверить `ADMIN_PASSWORD` в `admin-config.php` (после правки не забыть про кеш PHP) |
| `Требуется авторизация` после входа | Сессия протухла. Проверить `session.save_path` на сервере |
| `Битрикс24: ...` ошибка | Проверить `WEBHOOK_URL` в `config.php`, что вебхук жив |
| Поля не подгружаются, пустой экран | Открыть DevTools → Console, смотреть fetch-ошибки |
| Правки не сохраняются | Проверить права на запись в папку (для `admin-changes.log`) |
